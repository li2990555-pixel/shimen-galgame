# 头脑风暴：小红书矩阵号AI写作风格工具

## 产品核心定位

面向**小红书矩阵号博主**的AI写作风格引擎，解决多账号运营中风格一致性问题。

## 目标用户画像

| 维度 | 描述 |
|------|------|
| 用户类型 | 小红书博主，已有成熟个人号和稳定内容风格 |
| 运营模式 | 正在或准备做矩阵号（2-10个账号） |
| 当前工具 | 已在使用 ChatGPT/Claude 等通用AI工具 |
| 核心痛点 | 多账号口吻不一致，读者/平台能识别出非本人写作 |

## 用户三大痛点

1. **效率瓶颈** — 博主亲自写所有号太累，想用AI代笔但担心口吻不一致
2. **风格走样** — 用通用AI工具写，每个号语气飘忽，读者一眼看出不是本人
3. **素材复用** — 已有大量历史笔记，想基于它们批量生成新内容

## 产品三大核心功能

| 功能 | 描述 | 对应痛点 |
|------|------|---------|
| 风格提取 | 从博主已有笔记中提炼其独特的写作风格（用词、语气、句式结构、标点习惯等） | 风格走样 |
| 风格生成 | 基于提取的风格，将新想法/关键词生成完整笔记 | 效率瓶颈 |
| 风格改写 | 将他人/品牌的文案，用自己的风格重写 | 素材复用 |

## 关键假设

1. 小红书博主的写作风格是可量化、可提取的（篇幅较短，风格特征明显）
2. 博主愿意将自己的内容交给AI训练风格模型（信任成本可控）
3. 矩阵号场景付费意愿高（本身就在投入运营成本）

## 关键风险

1. 风格提取的精度 — 如果生成内容"不像我"，产品价值归零
2. 小红书平台对AI生成内容的政策风险
3. 竞品快速跟进（已有通用工具在做类似功能）

## 下一步建议

1. 需求调研 — 验证假设，深访5-10位矩阵号博主
2. 市场分析 — 竞品扫描，市场规模估算
3. 定义MVP范围
