# 卷王对决 V2 — 纯异步竞技设计

> 关键决策：去掉所有实时同步元素，专注时完全不被打扰
> 参考锚点：The Bazaar（异步自走棋）

---

## 一、为什么纯异步更优

```
同步模式的问题                         异步模式的优势
─────────────────                    ─────────────────
专注中看到对手进度 → 分心             专注时看不到任何对手 → 零干扰
需要两人同时在线 → 使用门槛高          各自找时间 → 灵活自由
实时状态同步 → 服务器+网络开销高       仅需提交专注数据 → 轻量
战中看到对手中断 → 自己也会松懈        专注后才知道结果 → 悬念+仪式感
```

**一句话**：学习工具的核心价值是帮用户学进去——异步对决把"干扰项"彻底移出专注时间。

---

## 二、新的核心循环

```
┌─────────────────────────────────────────────────┐
│                  异步竞技循环                      │
│                                                   │
│  备战                   专注                   结算 │
│  ────                  ────                   ──── │
│  选模式                 纯番茄钟                KO │
│  定计划                 无任何对手信息           数据对比 │
│  下赌注                 无任何游戏元素           段位变动 │
│  ↓                      ↓                       ↓    │
│  ┌────┐              ┌────┐              ┌────┐   │
│  │备战 │  专注25min   │ 专注 │  结算揭晓   │结算 │   │
│  │15s │ ───────────→ │25min│ ───────────→ │ 8s │   │
│  └────┘              └────┘              └────┘   │
│                                                   │
│  整个过程中，你完全不知道对手的实时数据               │
│  唯一知道的是：有一个段位相近的人和你同时在学          │
│  紧张感在结算那一刻集中释放                          │
└─────────────────────────────────────────────────┘
```

### 和 The Bazaar 的对比：

| The Bazaar | 卷王对决 |
|-----------|---------|
| 你 build 你的面板 | 你安排你的学习计划 |
| 你不知道对手在 build 什么 | 你不知道对手今天状态如何 |
| 战斗自动执行，你不操作 | 专注自动进行，你只负责学 |
| 战斗结束揭晓结果 | 番茄钟结束揭晓数据 |
| 结果由 build 质量决定 | 结果由专注质量决定 |

## 三、两种异步对战模式

### 模式 1：好友战书

```
小明 → 选择好友小红 → 下战书（25min / 50币）
       ↓
小红收到推送："小明向你发起挑战！25min专注，赌注50币"
       ↓
小红接受 → 双方在72小时内各自完成一次专注
       ↓
双方都完成后 → 系统对比 → 结算
       ↓
赢家拿走赌注，输家扣币
```

**特点**：
- 好友间的社交驱动力（不想输给朋友）
- 72 小时窗口，时间自由
- 赌注增加分量

### 模式 2：天梯匹配

```
用户点击「排位赛」→ 系统匹配段位相近的对手
       ↓
匹配到对手最近一次的专注数据（Ghost）
       ↓
用户开始专注 → 25/45/90 min → 完成
       ↓
系统自动和对手的 Ghost 数据对比
       ↓
赢了加分，输了扣分
```

**特点**：
- 无需好友在线，随时可打
- 匹配时间 < 1s（因为匹配的是历史数据）
- Ghost 是最佳异步机制（The Bazaar 已验证）

## 四、专注界面重新设计（彻底纯净）

```
┌──────────────────────────────┐
│                              │
│          25:00               │  ← 大字计时器，仅此而已
│                              │
│      第 1 个番茄钟            │  ← 进度标识
│                              │
│    ████████░░░░░░░░░░░░      │  ← 环形/线性进度
│                              │
│  ┌──────────┐ ┌──────────┐  │
│  │  暂停    │ │  结束    │  │  ← 极简控制
│  └──────────┘ └──────────┘  │
│                              │
│  📋 今日任务     ▼           │  ← 可折叠，默认为收起
│                              │
└──────────────────────────────┘

👆 这就是专注时的全部界面

没有任何对手信息
没有任何游戏元素
没有排行榜
没有段位提示
没有下注信息
没有任何闪烁/动画/通知
```

**为什么任务清单默认为收起**：因为真正专注的人不需要看任务清单——任务清单是战前设定的，战中看它说明你在分心。

## 五、结算揭晓（紧张感集中释放）

专注结束后，系统弹结算——**这时候才告诉你匹配到了谁、结果如何**：

```
┌──────────────────────────────┐
│                              │
│        📊 对战结果揭晓         │  ← 缓入动画，制造悬念
│                              │
│  ┌─────┐       ┌─────┐      │
│  │ 你   │       │ 影子 │      │  ← 对手匿名/昵称
│  │ 25min│       │ 22min│      │
│  │ 3🍅  │       │ 2🍅  │      │
│  │ 0中断│       │ 2中断 │      │
│  └─────┘       └─────┘      │
│                              │
│        🏆 你赢了！            │  ← 金色大字
│                              │
│  +15 段位分    -8 段位分      │
│  +50 专注币    -50 专注币     │
│                              │
│  对手段位：白银II（比你高一段）│  ← 匹配的是更强的对手
│  战胜高段位对手：额外 +5 分    │
│                              │
│  [再来一局]  [查看详细数据]   │
│                              │
└──────────────────────────────┘
```

**悬念设计**：
- 专注全程不知道对手是谁、什么段位、什么状态
- 结算时揭晓对手段位——如果匹配到高段位对手，赢了有额外加分
- 这就像 The Bazaar 里你打完才知道对手是什么 build

## 六、下战书功能（好友社交）

```
小明完成了一次 45min 专注，觉得自己状态很好
       ↓
点击「下战书」→ 选择好友小红
       ↓
系统用小明这次专注数据生成一个 Ghost
       ↓
小红收到推送："小明学了你 12 分钟，敢应战吗？"
       ↓
小红点开 → 看到小明的数据（时长、番茄钟、中断次数）
       ↓
小红接受 → 小红需要在 72h 内完成一次专注
       ↓
小红完成后 → 系统对比 → 结算
```

**这个设计的巧妙之处**：
- 下战书本身是一次学习成果的展示（"我学了 45min 无中断，你来破"）
- 好友的 Ghost 成为一个"靶子"——你知道要达到什么水平才能赢
- 社交裂变自然发生——"被我打败了？再打回来啊"

## 七、影子挑战（单人异步）

```
每次专注结束后，你的数据被保存为 Ghost
       ↓
下次专注你可以选择「影子挑战」
       ↓
系统匹配你自己上次的 Ghost
       ↓
专注完成 → 和过去的自己对比
       ↓
赢了：证明你在进步
输了：状态的波动，下次再战
```

**这是长期的留存核心**——当你没有好友在线、不想打排位时，和过去的自己比永远有意义。

## 八、设计变更总结

| 之前的设计 | 新的设计 |
|-----------|---------|
| 同步对战（同时在线） | **异步对战（Ghost 快照）** |
| 专注中看到对手状态和进度 | **专注中零对手信息** |
| 实时 WebSocket 同步 | **无需实时通信** |
| 战中"番茄冲击"（攻击动画） | **专注结束后统一结算** |
| 好友对战需要同时在线 | **下战书，72h 内回应** |
| 匹配需要对方在线 | **匹配历史 Ghost，秒匹配** |

### 之前同步设计中保留的元素

```
✅ 下注系统（赌注/专注币） → 保留，异步同样适用
✅ 段位系统（ELO/8段位）  → 保留，结算时计算
✅ 场馆/主题             → 保留，只影响战前战后视觉
✅ 放狠话                → 保留，下战书时可以留言
✅ 三种专注模式（25/45/90）→ 保留
✅ KO 结算动画            → 保留
✅ 战报分享               → 保留
✅ 赛季系统               → 保留
❌ 战中对手状态条          → 移除
❌ 实时状态同步            → 移除
❌ 休息时间的"攻击"反馈    → 移除（结算时统一展示）
```

### 技术架构变化

```
之前：需要 WebSocket 实时同步 → 架构复杂
现在：只需要 REST API     → 架构大幅简化

之前：需要两人同时在线 → 匹配逻辑复杂
现在：匹配 Ghost 快照  → 秒匹配，无等待

之前：专注中要处理状态推送 → 服务器压力大
现在：专注中无需任何推送  → 服务器极轻量
```

## 九、从 Bazaar 还给我们的核心启示

The Bazaar 证明了一个道理：

> **最好的异步竞技，让用户感觉"对手就在那里"，但又从不需要和对手同时在线。**

这是学习工具最理想的竞技形态——
- 需要陪伴感（知道有很多人同时在学）
- 需要竞技感（想赢）
- 但又不能真的被对手干扰（专注至上）

异步 Ghost 机制完美满足这三个需求。
