# MVP定义：小红书矩阵号写作风格工具

## MVP范围

**第一版只做：风格提取 + 风格生成**（风格改写放P2）

## 用户旅程

```
注册/登录
  │
  ▼
上传历史笔记（5-10篇，支持复制粘贴或上传txt）
  │
  ▼
风格分析 → 生成「风格报告」
  ├─ 用词特征：高频词、语气词、emoji使用习惯
  ├─ 句式偏好：短句/长句比例、标点习惯
  ├─ 语气倾向：亲切/专业/幽默/感性
  └─ 结构模式：开头方式、段落节奏、结尾习惯
  │
  ▼
用户审阅风格报告 → 可以手动调整（如"减少emoji使用"）
  │
  ▼
输入新笔记主题/关键词
  │
  ▼
生成笔记 → 标题(3选) + 正文 + 标签推荐
  │
  ▼
人工编辑微调 → 复制/导出
```

## MVP功能清单

### 核心功能

| 模块 | 功能 | 说明 |
|------|------|------|
| 风格提取 | 文本分析 | 上传5-10篇已有笔记 |
| 风格提取 | 风格报告 | 输出可视化风格分析结果 |
| 风格提取 | 参数调整 | 用户可手动微调风格参数 |
| 风格生成 | 输入主题 | 输入想法/关键词 |
| 风格生成 | 生成笔记 | 标题+正文+标签，符合风格 |
| 风格生成 | 多版本 | 同一主题生成2-3个版本可选 |

### 产品功能

| 模块 | 功能 | 说明 |
|------|------|------|
| 账号 | 注册登录 | 邮箱/手机号 |
| 账号 | 风格管理 | 保存多个风格Profile |
| 账号 | 历史记录 | 生成历史可回看 |
| 导出 | 一键复制 | 复制到剪贴板 |

## 明确不做（非MVP）

- ❌ 图片/封面生成
- ❌ 自动发布到小红书
- ❌ 矩阵号管理面板（多账号切换放后面）
- ❌ 风格改写功能
- ❌ 多人协作/团队版
- ❌ 爆款选题推荐
- ❌ 数据分析和复盘

## 成功标准

- 用户上传5篇笔记后，能在3分钟内完成风格分析
- 生成的笔记，用户认为"像自己写的" ≥ 70%
- 从输入主题到生成初稿 ≤ 30秒
- MVP开发周期：4-6周

## 技术方案方向

### 前端
- Web应用（Next.js/React）
- 支持文案输入、风格报告展示、生成结果编辑

### 后端
- 风格提取：LLM + 规则引擎（提取用词模式、句式特征）
- 风格生成：基于提取的风格特征构造Prompt，调用LLM API
- 用户风格Profile存储

### 关键设计点
- 风格提取不是训练一个模型，而是通过分析+Prompt工程实现（MVP阶段）
- 风格Profile可序列化为结构化数据，便于后续迭代优化
