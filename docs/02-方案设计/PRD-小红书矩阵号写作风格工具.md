# PRD：小红书矩阵号AI写作风格工具

**版本：** v1.0  
**状态：** Draft  
**优先级：** P0（MVP）

---

## 1. 产品概述

### 1.1 产品定位
面向小红书矩阵号博主的AI写作风格引擎。核心价值：让AI生成的内容"像你写的"，解决多账号运营中风格一致性问题。

### 1.2 产品 slogan
**"AI写的，但像你写的。"**

### 1.3 核心价值主张
- **对博主：** 从"亲自写所有号"到"定义一次风格，多号复用"
- **对矩阵运营：** 消除"AI代笔一眼假"的信任危机
- **对品牌：** 实现多账号内容标准化生产

---

## 2. 目标用户与用户故事

### 2.1 用户画像

**人物A：成熟博主（主账号10万+粉）**
- 年龄：25-35岁
- 领域：美妆/穿搭/家居/母婴
- 痛点：想做矩阵号但分身乏术，招人写又不像自己
- AI使用：用过ChatGPT，觉得"AI味太重"
- 付费意愿：愿意为高质量工具付费 200-500元/月

**人物B：矩阵号运营者（管理5-20个账号）**
- 身份：MCN/品牌运营
- 人群画像：25-30岁，新媒体运营
- 痛点：多个账号风格管理混乱，内容产出效率低
- AI使用：重度AI用户，使用多种工具
- 付费意愿：企业付费，500-2000元/月

## 3. 功能需求

### 3.1 风格提取模块（F1）

**F1.1 笔记上传**
| 项目 | 说明 |
|------|------|
| 输入方式 | 支持纯文本粘贴 + txt文件上传 |
| 数量要求 | 最低3篇，推荐5-10篇 |
| 每篇长度 | 不限，建议100-2000字 |
| 格式要求 | 纯文本，去除图片/格式标记 |

**F1.2 风格分析引擎**
分析维度：
- **用词特征：** 高频词、语气词（啦/呢/嘛/呀）、网络用语、专业术语密度
- **句式偏好：** 平均句长、短句/长句比例、问句/感叹句使用频率
- **语气调性：** 亲切/专业/幽默/感性/犀利（多维度打分）
- **emoji使用：** 使用频率、常用emoji列表、使用位置（句首/句中/句末）
- **内容结构：** 开头模式（痛点引入/故事开场/直给观点）、段落节奏、结尾模式（引导互动/总结升华/下期预告）
- **标点习惯：** 感叹号频率、省略号使用、分段偏好

**F1.3 风格报告展示**
- 可视化风格卡片，一目了然
- 每个维度附带示例（从用户笔记中提取的典型句子）
- 可编辑风格标签（如用户觉得"亲切"不对，可手动改为"幽默"）

**F1.4 风格参数调整**
- 用户可拖动滑块微调各维度（如"emoji使用：少 ←——→ 多"）
- 支持"排除样本"（某篇笔记不想让AI学习）
- 调整后实时预览风格效果

### 3.2 风格生成模块（F2）

**F2.1 输入主题**
| 项目 | 说明 |
|------|------|
| 输入方式 | 自由文本输入 |
| 可选字段 | 主题描述（必填）、补充素材（选填）、字数要求（选填） |
| 示例输入 | "推荐3款适合干皮的秋冬面霜" |

**F2.2 笔记生成**
- 生成内容：标题（3个选项）+ 正文 + 标签（3-5个）
- 生成方式：基于已提取的风格Profile构造Prompt，调用LLM
- 可配置参数：字数范围、语气强调（如"再口语化一点"）
- 生成时间：≤15秒

**F2.3 结果展示与编辑**
- 标题选项横向展示，点击即切换
- 正文支持在线编辑（富文本）
- 一键复制（标题 + 正文 + 标签 分组复制）
- 不满意可重新生成

### 3.3 账号与数据管理

**F3.1 用户系统**
| 功能 | 说明 |
|------|------|
| 注册 | 邮箱注册 |
| 登录 | 邮箱密码/验证码 |
| 账户管理 | 修改密码、注销 |

**F3.2 风格管理**
- 每个账户可创建多个风格Profile（P1，MVP可只支持1个）
- 每个Profile展示创建时间、笔记数量
- 支持重命名、删除

**F3.3 历史记录**
- 每次生成的内容自动保存
- 按时间倒序排列
- 支持搜索和筛选

## 4. 非功能需求

| 维度 | 要求 |
|------|------|
| 性能 | 风格分析 ≤ 3分钟（5篇500字笔记） |
| 性能 | 内容生成 ≤ 15秒 |
| 可靠性 | 服务可用率 ≥ 99.5% |
| 安全 | 用户笔记数据加密存储，不用于模型训练 |
| 安全 | 不允许用户上传他人作品分析（需本人原创） |
| 合规 | 生成内容需标注"AI辅助创作"选项（符合平台规范） |

---

## 5. 信息架构

```
首页
 ├── 创建风格（上传笔记 → 分析 → 报告 → 调整）
 ├── 生成笔记（选择风格 → 输入主题 → 生成 → 编辑 → 导出）
 ├── 我的风格（列表/详情/编辑）
 └── 历史记录
```

## 6. 成功指标

| 指标 | 目标值 | 测量方式 |
|------|--------|---------|
| 风格分析完成率 | ≥80%（开始分析的用户完成全流程） | 后端埋点 |
| 风格满意度 | ≥70%用户认为"像我写的" | 生成后问卷 |
| 用户留存（次日） | ≥40% | 登录记录 |
| 月人均生成次数 | ≥15次 | 后端统计 |
| 付费转化率 | ≥5%（参考行业数据） | 支付数据 |
