```

                       │ HTTP/REST

│  PostgreSQL  │            │    LLM API Layer    │
```

### 架构选型：全栈 Next.js

| 选项 | 选择 | 理由 |
|------|------|------|
| 前端框架 | Next.js 14+ (App Router) | SSR/SSG 支持，SEO友好，一体化部署 |
| UI库 | Tailwind CSS + shadcn/ui | 快速搭建，风格统一 |
| 数据库 | PostgreSQL (Neon / Supabase) | 结构化数据，JSON字段存储风格Profile |
| ORM | Prisma | 类型安全，Schema管理 |
| LLM API | Claude API (Anthropic) | 长文本理解能力强，风格分析任务表现优秀 |
| 部署 | Vercel | Next.js原生支持，零配置 |
| 文件存储 | Vercel Blob 或 AWS S3 | 用户上传的笔记原始文本 |

## 2. 风格提取引擎（核心技术）

### 2.1 设计思路

MVP阶段不训练模型，采用 **LLM + 结构化分析** 范式：

```
用户上传笔记
      │
      ▼
┌─────────────────────────────┐
│  Step 1: 单篇分析            │
│  每篇笔记独立提取风格特征      │
│  → 输出 JSON 结构数据         │
└─────────────────────────────┘
      │
      ▼
┌─────────────────────────────┐
│  Step 2: 多篇聚合            │
│  跨笔记统计 → 取交集/平均     │
│  识别稳定性高的特征           │
└─────────────────────────────┘
      │
      ▼
┌─────────────────────────────┐
│  Step 3: 风格Profile生成     │
│  结构化 JSON → 可读报告      │
│  存入数据库 + 展示给用户      │
└─────────────────────────────┘
```

### 2.2 风格Profile数据结构

```typescript
interface StyleProfile {
  id: string;
  userId: string;
  name: string;
  createdAt: Date;
  updatedAt: Date;

  // === 分析维度 ===
  vocabulary: {
    frequentWords: string[];           // 高频词列表
    toneWords: string[];               // 语气词（啦/呢/嘛/呀）
    internetSlangFreq: 'low' | 'medium' | 'high';
    jargonDensity: 'low' | 'medium' | 'high';
    uniquePhrases: string[];           // 标志性短语
  };

  sentence: {
    avgLength: 'short' | 'medium' | 'long';
    shortSentenceRatio: number;        // 短句占比 0-1
    questionFreq: number;              // 问句频率
    exclamationFreq: number;           // 感叹频率
    paragraphStyle: 'short_paragraphs' | 'long_paragraphs' | 'mixed';
  };

  tone: {
    friendliness: number;              // 0-1
    professionalism: number;           // 0-1
    humor: number;                     // 0-1
    emotionality: number;              // 0-1
    toneLabel: string;                 // "亲切闺蜜风" / "专业干货风" / ...
  };

  emoji: {
    frequency: 'rare' | 'moderate' | 'frequent';
    commonEmojis: string[];
    position: 'start' | 'middle' | 'end' | 'scattered';
  };

  structure: {
    openingPattern: string;            // "痛点引入" / "故事开场" / "直给观点"
    endingPattern: string;             // "引导互动" / "总结升华" / "下期预告"
    hasLists: boolean;
    hasSubheadings: boolean;
  };

  samples: {
    typicalOpening: string;
    typicalTransition: string;
    typicalClosing: string;
  };

  // 用户手动调整的覆盖项
  overrides?: Partial<StyleProfile>;
}
```

### 2.3 Prompt设计

**风格分析Prompt（单篇）：**

```
你是一位专业的写作风格分析师。分析以下小红书笔记的写作风格，
输出结构化的JSON特征数据。

笔记内容：
{note_text}

请从以下维度分析并输出JSON：
- vocabulary: 高频词、语气词、网络用语频率、标志性短语
- sentence: 句长偏好、问句/感叹句使用频率
- tone: 亲切度/专业度/幽默度/情绪度（0-1评分）
- emoji: 使用频率、常用emoji、使用位置
- structure: 开头模式、结尾模式、是否使用列表
- samples: 摘录1句典型开头、1句典型过渡、1句典型结尾

确保输出是合法的JSON。
```

**风格生成Prompt：**

```
你是一位小红书内容创作者。请按照以下风格Profile创作一篇新的小红书笔记。

【写作风格 Profile】
{vocabulary}
{sentence_style}
{tone}
{emoji_style}
{structure}

【典型表达样本】
- 开头风格：{sample_opening}
- 过渡风格：{sample_transition}
- 结尾风格：{sample_closing}

【本次创作主题】
{topic}

【补充素材（如有）】
{materials}

要求：
1. 标题：3个选项，20字以内
2. 正文：600-800字，分段自然
3. 标签：5个，#号格式
4. 严格遵循上述风格Profile
5. 避免使用"首先/其次/最后"等AI套话
6. 使用第一人称，口语化表达
```

## 3. 数据模型

```prisma
model User {
  id        String   @id @default(cuid())
  email     String   @unique
  password  String   // hashed
  createdAt DateTime @default(now())

  profiles  StyleProfile[]
  histories GenerationHistory[]
}

model StyleProfile {
  id        String   @id @default(cuid())
  userId    String
  name      String   @default("默认风格")
  rawData   Json     // 用户上传的原始笔记文本列表
  profile   Json     // 完整的风格Profile JSON
  status    String   @default("analyzing") // analyzing | ready | failed

  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  user      User     @relation(fields: [userId], references: [id])
  histories GenerationHistory[]
}

model GenerationHistory {
  id          String   @id @default(cuid())
  userId      String
  profileId   String
  topic       String   // 用户输入的主题
  title       String   // 生成的标题
  content     String   // 生成的正文
  tags        String[] // 标签列表
  userRating  Int?     // 用户评分 1-5

  createdAt   DateTime @default(now())

  user        User     @relation(fields: [userId], references: [id])
  profile     StyleProfile @relation(fields: [profileId], references: [id])
}
```

## 4. API设计

### 4.1 风格分析

```
POST /api/style/analyze
  请求: { notes: string[] }
  响应: { profileId: string, status: "analyzing" }
  说明: 异步分析，前端轮询状态

GET /api/style/{profileId}
  响应: { profile: StyleProfile, status: "ready" | "analyzing" | "failed" }

PATCH /api/style/{profileId}
  请求: { overrides: Partial<StyleProfile> }
  响应: { profile: StyleProfile }
```

### 4.2 内容生成

```
POST /api/generate
  请求: { profileId: string, topic: string, materials?: string }
  响应: {
    titles: string[],
    content: string,
    tags: string[],
    historyId: string
  }

POST /api/generate/retry
  请求: { historyId: string }
  响应: { titles: string[], content: string[], tags: string[] }
```

### 4.3 用户管理

```
POST /api/auth/register  { email, password }
POST /api/auth/login     { email, password }
GET  /api/auth/me        → 用户信息

GET  /api/history        → 生成历史列表
GET  /api/history/{id}   → 单条详情
```

## 5. LLM成本估算

| 操作 | Token消耗（约） | 成本（Claude Sonnet） |
|------|----------------|---------------------|
| 风格分析（每篇200字笔记） | ~1,000 tokens | ~$0.003 |
| 风格分析全套（5篇） | ~5,000 tokens | ~$0.015 |
| 风格生成（一篇笔记） | ~2,000 tokens | ~$0.006 |
| 用户月均50次生成 | ~100,000 tokens | ~$0.30 |

**单用户月成本：约 $0.50（按50次使用）**

**定价参考：$10-20/月 → 毛利率 95%+**

## 8. 技术风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|---------|
| LLM风格分析不一致 | 同笔记多次分析结果不同 | 温度参数设为0；多篇聚合取统计值 |
| 生成内容"不够像" | 用户流失 | 增加风格参数手动调整；加入样本句注入 |
| LLM API延迟 | 用户体验差 | 异步分析+轮询；生成时使用stream |
| 用户上传内容质量差（太少/太短） | 分析不准 | 最低3篇限制；引导用户提供高质量样本 |
