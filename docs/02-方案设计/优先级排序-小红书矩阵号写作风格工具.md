# 优先级排序：小红书矩阵号写作风格工具

## 功能清单

| # | 功能 | 描述 | 依赖 |
|---|------|------|------|
| F1 | 风格提取 | 从用户历史笔记中提取独特写作风格（用词、句式、语气、节奏、标点习惯等） | 无 |
| F2 | 风格生成 | 基于提取的风格，将新想法/关键词生成完整笔记 | F1 |
| F3 | 风格改写 | 将他人文案用自己的风格重写 | F1 |

## 依赖关系图

```
F1 风格提取 ──┬──→ F2 风格生成
              └──→ F3 风格改写
```

**风格提取是所有功能的前置条件。** 必须先做F1。

## 优先级评估矩阵

| 维度 | F1 风格提取 | F2 风格生成 | F3 风格改写 |
|------|------------|------------|------------|
| 用户价值 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| 技术难度 | 高（核心壁垒） | 中 | 中 |
| 稀缺性（竞品差距） | ⭐⭐⭐⭐⭐（最大） | ⭐⭐⭐ | ⭐⭐⭐ |
| 独立可用性 | ✅ 可独立交付 | ⚠️ 需F1 | ⚠️ 需F1 |
| 变现潜力 | 中（功能付费难） | 高（按量付费） | 高（按量付费） |

## 推荐优先级

### P0 — MVP必做
**F1 风格提取** — 这是产品的核心差异化和技术壁垒。无F1则F2/F3无法成立。
- 最小可行：用户上传5-10篇笔记 → 输出风格报告（用词特征、语气倾向、句式偏好）
- 可以作为一个独立的"写作风格诊断"产品交付

### P1 — 紧接迭代
**F2 风格生成** — 最能直接体现"省时间"价值，付费意愿最强。
- 输入：想法/关键词 + 已提取风格
- 输出：完整小红书笔记

### P2 — 第二迭代
**F3 风格改写** — 很有价值，但有潜在风险（改写他人作品可能涉及版权/平台判定问题）。
- 输入：他人笔记链接/文本 + 已提取风格
- 输出：用自己的口吻重写的版本

## MVP范围建议

**第一阶段：风格提取 + 风格生成**
```
用户上传笔记 → 风格分析 → 风格报告 ← 用户确认/调整
                                        ↓
                                  输入新想法 → 生成新笔记 → 人工微调 → 发布
```

**第二阶段：增加风格改写**
```
他人作品 → 风格改写 → 用自己的口吻输出 → 人工微调 → 发布
```

## 关键决策点

1. 风格提取的精度标准 — 用户觉得"够像"的阈值是多少？
2. 风格是否支持编辑/调整？还是完全黑盒？
3. 是否要支持多人协作（主号定义风格，子号使用）？
