## 一、产品概述

### 1.1 一句话描述

> 你看到有人在公共场合吸烟，你不想直接吵架，你按下这个盒子——它替你咳嗽出来。

### 1.2 产品简介

咳一下是一个手持大小的硬件装置。你看到有人在楼道口、公交站、电梯间吸烟，你走过去（或在不远处），按下盒子上的按钮。盒子发出一段真实的咳嗽声——老人的、小孩的、哮喘病人的——让吸烟者意识到自己的行为正在影响附近的人。它不骂你，不说话，只是放出被压制的声音。

**核心区别：它不是自动的。它需要你做一个动作——看见、选择、按下。**

### 1.3 核心逻辑

输入：你观察到有人吸烟 + 你主动按下按钮
逻辑：每按一次 → 从声音库中随机播放一段音频
输出：持续5-10秒的被影响者真实声音

## 二、问题与机会

### 2.1 背景

公共场合二手烟是中国城市最普遍的"附近的难以忍受"之一。尽管多地出台了禁烟条例，但在楼道、电梯口、公交站、公园角落等半公共空间，吸烟行为仍然普遍存在。被吸烟者往往选择沉默——因为"抱怨不值得"、"说了也没用"、"不想制造冲突"。

这种沉默本身就是附近的结构的特征：被冒犯的一方没有发声的机制。

### 2.2 核心问题

**问题一：被吸烟者的声音被礼貌压制。**
在公共场合，非吸烟者很少对吸烟者说"请不要在这里抽烟"——不是因为不介意，而是因为"不想惹麻烦"、"说了可能吵架"、"没必要为了几口烟折腾"。这种沉默让吸烟者感知不到自己的行为对他人的影响。

**问题二：现有的禁烟标识已经失效。**
"禁止吸烟"的标识到处都是，但吸烟者已经对它们视而不见。文字标识是最弱的干预——它没有温度、没有处境、没有"人"的存在。

**问题三：公共场所缺少"说话的东西"。**
目前公共场所的电子装置（烟雾报警器、监控摄像头）都是"管理工具"——它们是权力装置，不是"附近的存在"。它们不会告诉你"这里有小孩在呼吸"。

**问题四：附近缺少非对抗的沟通中介。**
直接劝阻吸烟容易引发冲突。被吸烟者需要的不是一个吵架的工具，而是一个"替我说出我的不舒服"的中介物。

### 2.3 机会

1. **需求真实且高频**：公共场合吸烟是中重度城市病，几乎所有非吸烟者都有过被二手烟困扰的经历——以及"想说什么又算了"的时刻
2. **技术成熟且低成本**：AI语音模块（几十元）+ 按键 + 电池 = 一个打火机大小的盒子，成本50-120元
3. **社交传播潜力**：这个盒子的设计本身具有传播力——"一个替你咳嗽的盒子"，人们会拍照、讨论、分享
4. **竞品空白**：市面上没有"手持式替被影响者发声"的产品
5. **方向差异明显**：与回眸（流动空间感知）、家政女工互助平台（同行互助）形成完整的三方向叙事

## 三、目标用户

### 3.1 用户分类

| 角色 | 描述 | 关系 |
|------|------|------|
| **吸烟者** | 在公共场合吸烟的人 | 被干预的对象 |
| **被影响者** | 被动吸入二手烟的人 | 使用盒子的主体——按下按钮的人 |
| **旁观者** | 附近的路人 | 听到声音、感知到冲突的第三方 |

### 3.2 典型场景

**场景一：楼道口吸烟**
你在楼道口等电梯，旁边有人点了一根烟。你不想直接说"别在这抽"——你知道那可能引发冲突。你把手伸进口袋，摸到一个打火机大小的盒子，按了一下。"咳……咳……"一声老人的咳嗽从你身边发出——不响，但刚好让吸烟者听到。他回头看了你一眼，你没说话。他把烟掐了。

**场景二：公交站等车**
一个人在公交站吸烟。你站在不远处，犹豫了一下，从兜里拿出盒子，按了一下。盒子发出一段小孩的声音："妈妈，好呛。"那个人转头看了一眼，身边没有小孩。他看到了你手里的盒子。他没说话，走远了几步把烟灭了。

**场景三：电梯间**
电梯门开了，里面的一个人还在吸最后一口。你跟进去，按了一下口袋里的盒子——婴儿哭声 + 老人喘息声。那人愣住了，把烟藏到手心。你没看他，他也没看你。电梯到了，各走各的。

## 四、产品方案

### 4.1 硬件方案

**基础硬件（基于市售AI语音模块修改）：**

| 组件 | 规格 | 说明 |
|------|------|------|
| 主控芯片 | ESP32 / 类似AI语音模块主控 | 低功耗，支持WiFi联网，可OTA更新声音库 |
| 按钮 | 机械按键 | 核心输入——按下即播放，有明确的触感反馈 |
| 扬声器 | 2W-3W 小喇叭 | 音量约60-65dB（相当于正常说话音量，不刺耳但足够让对方听到） |
| 指示灯 | LED | 待机：灭；播放中：红色灯亮 |
| 电源 | 纽扣电池 / 锂电池 | 可随身携带，续航>1个月（按日均使用5次计） |
| 外壳 | 3D打印或成品开模 | 约一包纸巾大小，带挂绳孔（可挂钥匙扣/放口袋/挂包上） |

**成本估算：**
| 组件 | 单价（元） |
|------|-----------|
| 主控+AI语音模块 | 30-60 |
| 按键+指示灯 | 2-5 |
| 扬声器 | 3-8 |
| 电池 | 5-15 |
| 外壳（3D打印） | 10-30 |
| **总计** | **约50-120元/个** |

### 4.2 声音库设计

这是产品的核心内容。声音库中存储的音频片段由真人录制，所有录音者均为"日常被二手烟影响的无名者"。

**声音库类别：**

| 类别 | 内容 | 时长 | 录制来源 |
|------|------|------|---------|
| 老人咳嗽 | 老年人被烟呛到的咳嗽声 | 5-8秒 | 真实老人录制 |
| 小孩呛声 | 儿童吸入烟雾后的呛咳声 | 3-5秒 | 真实儿童录制 |
| 婴儿哭 | 婴儿被烟味刺激的哭声片段 | 5-10秒 | 环境录制 |
| 哮喘喘息 | 哮喘病人呼吸不畅的喘息声 | 5-8秒 | 真实哮喘患者录制 |
| 孕妇不适 | 孕妇被烟味刺激后的声音 | 5-8秒 | 真实孕妇录制 |
| 劝阻语 | 简短自然的劝阻（非攻击性） | 3-5秒 | 真人录制 |
| 混合组 | 以上多种声音的叠加剪辑（如老人咳嗽+小孩说"好呛"） | 10-15秒 | 声音编辑 |

**劝阻语音示例（C类灵感补充）：**
- "哎……这里有人"（老人声音，平静的）
- "妈妈这里好呛"（小孩声音）
- "能不能别在这抽"（年轻人声音，不带攻击性）

**声音库原则：**
- 所有声音均来自真实的人，非AI生成（不能是合成音）
- 不包含辱骂、攻击性语言
- 不包含"你"的直接指向（不是为了"被你"而是为了"让人听到"）
- 定期更新（通过WiFi连接，OTA更新声音库）

### 4.3 交互逻辑

```
[初始状态] 待机中，灯灭
    │
    ▼
你观察到有人吸烟
    │
    ▼
你从口袋/包里拿出盒子
    │
    ▼
按下按钮
    │
    ├── 短按（>0.3秒） → 随机播放声音库中的1段音频
    │       播放时长：5-10秒
    │       音量：60-65dB（正常对话音量）
    │       播放时长到自动停止，回到待机
    │
    ├── 连按2次（2秒内） → 播放混合组音频（多种声音叠加）
    │       播放时长：8-12秒
    │       适用于吸烟者距离较近或烟味特别重时
    │
    └── 长按（>3秒） → 不播放，进入"录制模式"
            （录制模式可用于后续扩展：录一段你自己的咳嗽声加入声音库）
```

**关键交互原则：**
- 按钮是唯一的交互方式——简单、直接、无需学习
- 每次按下只播放一段，不循环、不停续——你按一次，它替你说一次
- 你决定什么时候按、按几次——主动权在你自己手里
- 没有开关机概念，待机功耗极低

## 五、非功能需求

### 5.1 硬件需求

| 需求 | 说明 |
|------|------|
| 功耗 | 待机功率<0.01W（可用纽扣电池待机数月），发声功率<2W |
| 便携性 | 重量<80g，可挂钥匙扣/放口袋/挂包上 |
| 按键寿命 | 机械按键>10万次按压 |
| 工作环境 | -10°C至50°C，基本防尘 |
| 续航 | 纽扣电池版>3个月（日均使用5次），锂电池版>1周（可充电） |

### 5.2 内容需求

| 需求 | 说明 |
|------|------|
| 声音更新 | 通过WiFi/蓝牙连接手机App，远程更新声音库（平均每季度更新1次） |
| 音量固定 | 不设音量调节（避免被调至无声），出厂预设为60-65dB |
| 隐私保护 | 设备不录音不上传，不收集任何用户数据 |

### 5.3 安全需求

| 需求 | 说明 |
|------|------|
| 日常安全 | 外壳无锐角，电池防短路 |
| 内容安全 | 声音库中无攻击性、违法内容；出厂预装内容经过审核 |

## 六、3天实施计划

### 6.1 MVP定义

由于3天无法完成硬件量产，MVP的交付物为：
1. **功能原型**（1个）：基于市售AI语音模块改造，实现按钮按下→播放的核心逻辑，装入简易外壳
2. **声音库样本**：录制3-5条真实声音片段（老人咳嗽、小孩呛声、孕妇不适、哮喘喘息、劝阻语）
3. **场景演示视频**：拍摄一段约1分钟的场景演示（楼道口/公交站的真实场景模拟）
4. **路演演示**：现场演示原型+播放演示视频

### 6.2 实现计划

**Day 1：硬件原型搭建**

| 任务 | 说明 |
|------|------|
| 准备AI语音模块 | 用户分享的淘宝链接或类似市售产品 |
| 按键接入 | 将机械按键连接到模块的GPIO引脚，配置按下触发 |
| 烧录基础程序 | 按键检测→随机音频播放，无其他逻辑 |
| 制作简易外壳 | 3D打印或手工制作（能装下模块+按键+电池即可） |

**Day 2：声音库录制 + 整合**

| 任务 | 说明 |
|------|------|
| 录制声音样本 | 找3-5名志愿者录制咳嗽、呛声等音频片段（1天完成录制） |
| 音频剪辑和降噪 | 处理成适合扬声器播放的格式，裁剪至5-10秒 |
| 烧录声音到模块 | 将音频文件写入语音模块的存储芯片 |
| 外壳最终装配 | 将硬件装入外壳，配挂绳 |

**Day 3：测试 + 路演准备**

| 任务 | 说明 |
|------|------|
| 场景测试 | 在实际场景中测试按键→播放的完整流程，调整音量和触发手感 |
| 场景视频拍摄 | 拍摄一段演示视频（楼道口/公交站模拟） |
| 路演PPT | 三个方向合并为一个PPT |
| 概念文档定稿 | 方向二概念方案定稿 |

### 6.3 风险与缓解

| 风险 | 缓解方案 |
|------|---------|
| 硬件采购运输来不及（3天） | 备选方案1：用手机Web模拟（点击按钮→播放音频）；备选方案2：直接展示盒子设计稿+概念视频 |
| 声音录制质量不佳 | 使用手机自带录音在安静房间录制即可，不追求专业音质 |
| 按键手感不好 | 可以设计为"打火机手感"——按下去有咔哒声，增加仪式感 |
| 无法在3天内完成硬件开发 | 主打概念视频+设计稿+现场演示流程，硬件原型为加分项 |

> "这是一个咳一下。比打火机大一点。"

>
> "它只说这5秒。不循环、不重复。你按一下，它替你说一次。你决定什么时候按，按不按。"

| 名称 | 理由 |
|------|------|
| 咳/咳咳 | 直接、好记，就是那个声音 |

## 九、开放问题

1. **合法性问题**：在公共场合放置发声装置是否需要许可？会不会被当作"噪音扰民"投诉？MVP阶段建议以"艺术装置/社会实践"而非"商品"的形式展示。
2. **声音库版权**：所有人录制的音频素材如何处理授权？建议签署简单的知情同意书。
3. **硬件量产路径**：3天后如需继续推进，可以联系AI语音模块厂商定制化生产（已有现成硬件方案，只需加烟雾传感器和改固件）。
4. **被破坏风险**：放置在公共空间的硬件存在被破坏/盗窃风险。MVP阶段不考虑这个问题，但如果长期落地需要考虑防盗设计。
5. **伦理边界**："咳嗽声"会不会被当作噪音引起居民反感？会不会被吸烟者认为"管的太宽"？需要在实际测试中观察反馈。
