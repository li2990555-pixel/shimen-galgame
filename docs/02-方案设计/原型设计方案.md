## 一、设计概述

### 1.1 产品定位

一个基于"流动空间"的轻量日记体平台，让城市中擦身而过的陌生人在异步日记中相遇。

### 1.2 一句话介绍

> 写一段通勤路上的观察，AI会让你知道——有人和你在同一节车厢里想过类似的事。

### 1.3 核心洞察

城市人的附近正在从"定居空间"（小区/邻里）迁移到"流动空间"（地铁/路口/便利店）。我们每天和陌生人共处一室却从不交流——不是不想，是没有机制。这个产品提供一个轻量的"异步相遇"机制：你写你的日记，AI在数据中发现你和别人的连接，生成"如果你们相遇"的对话。

### 1.4 核心逻辑

1. 用户记录自己在流动空间中的观察和心情（文字或语音）
2. AI从日记中提取隐含的**空间线索**（地铁几号线/路口名称等）
3. 当两条日记在空间/主题上接近时，AI生成一段"如果相遇"的虚构对话
4. 用户在时间线上浏览日记和AI连接
5. 被故事打动后可选择联系作者

## 五、用户使用路径

1. 用户打开网页 → 进入卡片流（已有3-5条AI生成的样例日记）

### 第二天打开
1. 看到素材库有新连接标记 → 点进去
2. AI已自动在素材库中找到了日记之间的连接 → 阅读"相遇对话"

### 核心体验闭环

滑卡片 → 收集故事 → AI发现连接 → 阅读相遇 → 被触动 → 自己写 → 回到滑卡片

## 七、AI提示词设计

### 核心提示词：生成相遇对话

```
你是一个城市观察者。以下两条日记来自同一个空间（{space}），
由不同的人在不同时间写下。

日记A：{contentA}
日记B：{contentB}

请生成一段80-150字的虚构对话，场景设定为：
如果这两个人在{space}中擦肩而过、对视了一眼。
对话应该：
1. 自然、日常、不煽情
2. 从两篇日记的共同点出发（比如都注意到了同一件事）
3. 保持克制，不要过度戏剧化
4. 对话结束后仍保持陌生人的距离感（可能只是对视、微笑、然后各走各的）

只输出对话本身，不要加任何说明。
```

### 连接发现逻辑（素材库内）

AI连接不是全局的，而是**基于每个用户的素材库**独立生成：

1. 当用户的素材库内新增第N条日记（N≥2），触发一次连接扫描
2. 扫描范围：仅限该用户素材库内的全部日记
3. 扫描逻辑：查找space相同 / 主题相近的日记对
4. 对每个候选对调用上述提示词生成对话
5. 连接展示在素材库页面的"AI相遇"区域

**为什么是基于素材库而非全局：**
- 全局连接是"所有人之间"的，但用户不关心不相关的连接
- 素材库内的连接是用户**自己筛选的**——你收集了这些故事，你自然想看它们之间有什么关系
- 这给了用户收集的动机：收集越多，AI能发现的连接越多
